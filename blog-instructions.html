Adding a Blog Post

Follow these steps each time you want to publish a new post. The only files you need to touch are a new .md file and any media assets. The build script handles the rest.

Step by step

1

Create a new markdown file

Add a file to blog/posts/ named with the post slug. Use lowercase with hyphens, no spaces.

blog/posts/why-regulation-matters.md
2

Add frontmatter at the top

This block tells the build script the post's metadata. All fields are required.

---
title: Why Regulation Matters
date: 2026-03-15
slug: why-regulation-matters
excerpt: Most pianists only think about tuning, but regulation is what makes a piano feel alive under your fingers.
tags: [regulation, technique]
---
3

Write the post body in markdown

Standard markdown below the frontmatter block. You can embed audio, video, or any HTML directly inline.

Most pianists only think about tuning...

## The action explained

Each key triggers a chain of parts called the action...

<!-- Embed an audio clip -->
<audio controls src="../media/before-after.mp3"></audio>

<!-- Embed a video -->
<video controls src="../media/action-demo.mp4" style="width:100%"></video>

<!-- Embed a YouTube video -->
<iframe width="100%" height="400"
  src="https://www.youtube.com/embed/VIDEO_ID"
  frameborder="0" allowfullscreen></iframe>
Media files Place audio and video files in a blog/media/ folder and reference them with a relative path from the post template: ../media/filename.mp3
4

Run the build script

From the project root in Terminal:

node build.mjs

This will regenerate blog/posts.json and write the post's HTML output file.

5

Preview locally

Start the local server if it isn't running:

node serve.mjs

Then open http://localhost:3000/blog.html to see the listing, and click through to the new post.

6

Commit and push

When the post looks right, commit everything and push to publish.

git add .
git commit -m "Add post: why-regulation-matters"
git push

Frontmatter reference

  • title — The post heading as it appears on the listing and post pages.
  • date — Publication date in YYYY-MM-DD format. Used for sorting.
  • slug — URL-safe identifier. Must match the filename (without .md).
  • excerpt — One or two sentences shown on the listing page. Write this yourself — don't let it auto-generate.
  • tags — Optional list for future filtering. Use lowercase, e.g. [tuning, regulation].

File structure

blog/
  posts/
    why-regulation-matters.md   ← you write this
    another-post.md
  media/
    before-after.mp3            ← audio/video assets go here
  posts.json                    ← auto-generated by build.mjs
blog.html                       ← listing page
blog/post.html                  ← post template (single file, reads slug from URL)
