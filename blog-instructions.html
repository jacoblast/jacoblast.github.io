Adding a Blog Post

The blog runs on Jekyll, the same system that builds the rest of the site. Adding a post means creating one markdown file and committing it — Jekyll handles the rendering automatically.

Step by step

1

Create a new markdown file in _posts/

File names must follow Jekyll's date-slug format exactly. Use lowercase with hyphens, no spaces.

_posts/2026-05-10-why-regulation-matters.md
2

Add front matter at the top

This block tells Jekyll how to render the post. All fields are required.

---
layout: post
title: "Why Regulation Matters"
date: 2026-05-10
description: "Most pianists only think about tuning, but regulation is what makes a piano feel alive under your fingers."
excerpt: "Most pianists only think about tuning, but regulation is what makes a piano feel alive under your fingers. Here's what it is and why it matters."
tags: [regulation, technique]
permalink: /blog/why-regulation-matters/
---
description vs. excerpt description goes in the <meta> tag — keep it under 160 characters, no trailing punctuation.

excerpt appears on the /blog listing page. Can be one or two sentences — write it yourself rather than letting it auto-generate from the first paragraph.
3

Write the post body in markdown

Standard markdown below the front matter block. Use ## for section headings (they render as <h2> in the post layout). You can embed HTML inline if needed.

Most pianists only think about tuning...

## The action explained

Each key triggers a chain of parts called the action...

  frameborder="0" allowfullscreen style="margin:32px 0"></iframe>
4

Preview locally

If Jekyll isn't already running, start it from the project root in Terminal:

export PATH="/opt/homebrew/opt/ruby@3.3/bin:/opt/homebrew/lib/ruby/gems/3.3.0/bin:$PATH"
bundle exec jekyll serve --watch --host 0.0.0.0 > /tmp/jekyll.log 2>&1 &

Then open http://localhost:4000/blog/ to see the listing, and click through to the post.

Already running? Jekyll rebuilds automatically on file changes when started with --watch. No need to restart. Allow a couple of seconds after saving.
5

Commit and push to publish

When the post looks right, use the Claude Code skills to commit and push:

/commit
/push

GitHub Actions will rebuild the site automatically. The post will be live within a minute or two.

Front matter reference

  • layout — Always post. This uses _layouts/post.html.
  • title — The post heading. Used on the listing page, post page, and in <title> / Open Graph tags.
  • date — Publication date in YYYY-MM-DD format. Must match the filename prefix. Used for sorting on the listing page.
  • description — One sentence for the meta description tag. Under 160 characters.
  • excerpt — One or two sentences shown on the listing page. Write this yourself.
  • tags — Optional list, e.g. [tuning, regulation]. Lowercase.
  • permalink — The URL path for the post. Use /blog/the-post-slug/.

File structure

_posts/
  2026-04-28-how-often-should-you-tune-your-piano.md   ← posts go here
  2026-05-10-why-regulation-matters.md

_layouts/
  default.html     ← site shell (header, footer, scripts)
  post.html        ← post page layout (extends default)

blog.html          ← /blog listing page (auto-reads site.posts)
